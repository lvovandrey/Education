Главные команды.
Clone - скачиваем репозиторий удаленный в свой локальный
Commit - сохраняем изменения файла в локальном репозитории
Push - закачиваем сохраненные в локальном репо коммиты в удаленный репозиторий

Откат изменений.
Discard - Ctrl+Z - убираем изменения незакомиттеного файла
Revert/Revert&Commit - откатываем изменения в локальном репозитории.
Reset - hard - Убираем все до пуша на удаленном репозитории

Ветви.
Add branch - добавляет ветвь разработки. Причем новая ветвь - фактически новый набор файлов, который 
клонируется из родительской ветки.
Merge - переносит все изменения из одной ветви в другую.  
порядок операций с мердж такой:
1. Делаем Мердж для теста - сливаем из родительской ветки все изменения в новую ветку, 
2. Тестируем полученную новую ветку - что сломалось смотрим.
3. Когда все починилось - делаем окончательный merge уже обратно - в родительскую ветку вносим
изменения из новой ветки. Полученный коммит можно уже запушить на удаленный репо.

Кстати новую ветку можно вообще не пушить, можно с ней работать чисто в локалном репо.

Конфликты


Cherry-Pick - отдельный коммит из одной ветки можно залить в другую.
С ЧЕРИПИК НАДО ЕЩЕ ПОПРОБОВАТЬ - ПЛОХО РАЗОБРАЛСЯ

Ошибки
1. Боязнь создания новых коммитов
2. Добавление в репозиторий бинарников и exe
3. В ветви которую мы редактируем должны быть все файлы репозитория
4. Делать очень большие коммиты
5. В разных ветках не должны дублироваться изменения. Иначе возникаю конфликты

С консолью
Как-то все непросто поначалу кажется
Но разобраться в целом можно но сложно

Теги
Можно пометить комит тегом, а потом создать отдельную ветвь идентичную состоянию в этом 
коммите- для этого тыкаем в Tags - выбираем создание ветви
